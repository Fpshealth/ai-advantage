# AI-Advantage

### Cowork: Team-AI — turn everyday work into an AI-native team

Skills that help you turn your work into clean, reusable documents. **One task = one chat = one
finished file.** You start a skill by writing what you need in a new chat — Claude recognizes the
right skill automatically. Works in **Claude Cowork** and **Claude Code** alike.

This is a real, working example of what "AI-native" actually looks like day to day — not a demo
built to impress, the same full-depth toolset a team uses every day: documenting how work gets
done, questioning whether it should be done that way at all, and building in a quality check at
every handoff so nothing quietly drifts.

> **Set up once, then go.** Before you work for the first time, set up your safe workspace (see
> **Sandbox Setup** below). You do this **only once** — after that, every other skill puts its
> files in the right place automatically.

| Skill | For what | Example trigger |
|---|---|---|
| **Sandbox Setup** | One-time setup: creates your safe workspace (`01_Input` / `02_Work` / `03_Output`) and the `CLAUDE.md` rules file Claude follows. | *"Set up my workspace"*, *"set up the sandbox"* |
| **Prompt Improvement** | Claude's answer wasn't helpful? We fix your prompt — and you learn how a good prompt is built along the way. | *"The answer doesn't help me"*, *"improve my prompt"* |
| **SOP Creation** | Capture one task step by step as instructions a new colleague could follow directly. | *"Create an SOP"*, *"document this task"* |
| **Process Documentation** | Capture a whole process across roles — the overview for the manager. | *"Document a process"*, *"map the end-to-end process"* |
| **Process Exploration** | **Rethink** a process instead of just capturing it: from a rough idea to a clear target picture, every step questioned, tagged for AI-readiness at the end. | *"Explore a process"*, *"rethink this process"*, *"design the target process"* |
| **Visualizer** | Turn an SOP or a Process Documentation into a clean HTML view for checking. | *"Visualize this"* |
| **Escalation** | Stuck? Send the whole session to your consultant in two clicks. | *"Escalate"* |

**How handoffs work:** everything is a **file in your workspace** — never copied out of the chat.
Drafts land in `02_Work`, finished results in `03_Output` (set up by the sandbox skill). For the
next step, open a **new chat in the same folder** and start the next skill; it finds the file
automatically.
